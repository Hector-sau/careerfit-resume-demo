# AI/大模型产品实习简历优化 Demo

## 说明

本 Demo 用于展示“原始经历如何被优化成更贴近 AI 产品实习的简历表达”。

参考 JD 来自腾讯招聘公开岗位“AI产品经理-AI生成游戏”。该岗位不是实习岗，因此本 Demo 不把它当作实习岗位原文，而是用它反推 AI/大模型产品实习需要准备的能力。

来源链接：http://careers.tencent.com/jobdesc.html?postId=2011285787706019840

## 目标岗位 JD 摘要

岗位方向：AI 产品、AI 生成游戏、LLM 对话交互。

核心要求：

- 理解 AI/大模型应用场景
- 能围绕用户需求设计产品方向
- 能设计用户旅程和对话式交互流程
- 能编写 System Prompt、Few-shot Examples、需求澄清问题
- 能建立基础评测体系，通过测试案例优化 AI 输出
- 有 PRD、数据分析、产品文档能力加分

## 模拟用户原始背景

```text
本科商科专业，无互联网实习。
做过课程小组汇报，使用 ChatGPT 辅助搜集资料和整理报告。
运营过小红书账号，有 3000 粉丝，内容主要是学习经验和生活方式。
会 Excel、Canva，会基础数据整理。
想投 AI 产品实习，但不知道怎么把经历写到简历里。
```

## 简历问题诊断

原始表达的问题：

1. “使用 ChatGPT 辅助完成课程报告”太弱，不能体现 Prompt、信息整理或流程设计。
2. “运营小红书账号”没有体现用户洞察、选题、数据复盘。
3. 缺少 AI 产品相关项目，会让岗位匹配度偏低。
4. 没有 PRD、竞品分析、评测报告等可展示产出。

## 优化前

```text
使用 ChatGPT 辅助完成课程报告，提高写作效率。
运营小红书账号，发布学习和生活内容。
```

## 输出 A：优化后的简历

保守版，可以直接使用：

```text
- 使用 ChatGPT、Kimi 辅助完成课程调研与内容整理，尝试信息提取、摘要生成、结构化改写等 Prompt 流程，提升资料整理效率。
- 独立运营小红书账号，围绕学习经验与生活方式内容进行选题策划、标题优化和数据复盘，累计获得 3000+ 粉丝。
- 参与课程小组汇报，负责资料搜集、内容结构梳理和展示文稿优化，沉淀结构化表达和协作经验。
```

增强版，需要补充项目后再使用：

```text
- 基于 AI 产品实习岗位 JD，整理 10+ 岗位要求，提取高频能力标签，搭建 AI 求职路线规划 Prompt 工具包，支持岗位匹配、能力差距分析与学习路径生成。
- 设计 JD 解析、匹配评分、简历优化和项目补齐 4 类 Prompt 模块，并通过真实 JD 测试记录漏提取、误判和建议泛化等问题。
- 围绕 AI 求职小白场景，输出示例报告、个人背景模板和失败案例记录，降低非技术用户使用 AI 工具优化简历的门槛。
```

## 输出 B：JD 匹配建议（不直接写入简历）

下面内容是基于 JD 的求职建议，不是当前简历内容。只有当项目真实完成并有交付物后，才可以把对应表达写进简历。

### 可补齐项目建议

项目名：AI 求职路线规划工具包

目标：补齐 AI 产品实习需要的 JD 分析、Prompt 设计、评测意识和产品文档能力。

最小版本：

- 5 个 Prompt
- 3 个岗位 Demo
- 10 份 JD 测试记录
- 1 份失败案例文档
- 1 个 GitHub 仓库

项目产出物：

- README
- Prompt 模板
- 示例输出报告
- 评测方法
- 失败案例
- 小红书分享文案

完成后简历可写：

```text
面向 AI/大模型实习求职人群，设计 JD 解析与职业匹配 Prompt 工具包，覆盖岗位要求提取、技能矩阵生成、能力差距诊断、简历优化建议与能力补齐建议。
```

### 候选项目对比

| 候选项目 | 预计周期 | 适合选择的情况 | 不适合选择的情况 |
| --- | --- | --- | --- |

备选项目说明：体验 3-5 款 AI 产品/Agent 工具（例如 Kimi、豆包、Coze、Dify 搭建的应用），从用户旅程、对话设计、System Prompt 猜测角度做功能拆解，输出 1 份改进建议文档。比完整工具包轻量，适合先验证方向。

### 信息来源与判断依据

| 建议 | 信息来源 | 具体依据 | 可信度 | 需要用户核实什么 |
| --- | --- | --- | --- | --- |
| 建议优先补一个 AI 产品项目而非直接投递 | JD 原文 + 用户履历 | JD 要求 Prompt 设计和评测体系经验，用户仅有 ChatGPT 使用经历 | 高 | 无 |
| 匹配度总分 57/100 | 用户履历 + JD 解析结果 | 按 5 个维度加权计算，缺少产品化项目是主要扣分点 | 中 | 目标公司是否对该分数区间有明确的初筛线 |
| 小红书运营经历可迁移到用户洞察能力 | 用户履历 | 内容策划、选题和数据复盘与用户洞察能力相关 | 中 | 面试官是否认可跨领域经历的迁移逻辑 |

### 面试风险和注意事项

- 不能把“用过 ChatGPT”写成“熟悉大模型产品设计”。
- 不能凭空写 PRD、评测体系或用户访谈，必须真的产出文档后再写。
- 如果没有数据，不要编造提升百分比，可写“整理、设计、输出、复盘”等真实动作。
- 这个 Demo 更适合 AI 产品实习和 AIGC 运营，不适合包装成算法或模型训练经验。
