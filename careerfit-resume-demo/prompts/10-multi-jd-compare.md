# 10 多 JD 对比 Prompt

## 使用场景

求职小白经常同时看中 2-3 个岗位，但不知道该优先投哪个、同一份简历要不要分别改。这个模块用一份真实背景对比多个 JD，输出岗位要求对比表、匹配度排序、投递策略和每个岗位的简历调整点。

Resume-Matcher 的 roadmap 里有 "Multi-job description optimization"，但截至 2026-07 尚未落地；本项目用纯 Prompt 先实现了这个需求。

## 设计原则

- 最多对比 3 个 JD，再多会稀释分析质量。
- 每个岗位的匹配度都要给依据（JD 原文要求 + 用户对应经历），不允许只给分数。
- 排序是建议不是决定，明确"最终选择权在用户"。
- 简历差异化调整只调整表达重点，不为不同岗位编造不同经历。
- 涉及公司近况、薪资等信息，要求 AI 声明"需要补充公开信息后再判断"。

## Prompt 模板

```text
你是一名严谨的求职顾问。我在同时考虑 N 个岗位，需要你基于我的真实背景对比这些 JD，帮我判断优先投哪个。

重要原则：
1. 只基于我提供的背景判断，不允许编造我没有的经历。
2. 每个结论都要给出依据（来自哪个 JD 的哪条要求、或我的哪条经历）。
3. 不要为了让结论好看而抬高匹配度；差距大就直说。
4. 最终选择权在我，你只提供排序建议和理由。

【岗位 1 JD】
<<<
粘贴第 1 个 JD
>>>

【岗位 2 JD】
<<<
粘贴第 2 个 JD
>>>

（如有岗位 3，同格式继续）

【我的个人背景】
- 学历/专业：
- 目标岗位：
- 过往实习：
- 项目经历：
- 校园/社团经历：
- 内容/运营经历：
- 会使用的工具：
- AI 工具使用经历：
- 当前简历里想优化的内容：

请输出：

## 1. 岗位要求对比表

| 对比维度 | 岗位 1 | 岗位 2 | 岗位 3（如有） |
| --- | --- | --- | --- |
| 核心职责 |  |  |  |
| 必备技能 |  |  |  |
| 加分项 |  |  |  |
| 对我最大的门槛 |  |  |  |

## 2. 我的匹配度排序

按匹配度从高到低排序，每个岗位给出 0-100 分和三条具体依据（引用 JD 原文要求 + 我的对应经历）。

## 3. 投递策略建议

- 优先投递哪个，为什么
- 哪个可以同时投但期望要放低，为什么
- 哪个建议先补齐能力再投（如有），需要补什么

## 4. 同一份简历的差异化调整

如果我用同一份简历投这几个岗位，请分别指出针对每个岗位最需要调整的 2-3 处表达（只调整表达重点，不编造内容）。

## 5. 信息来源说明

关键判断请标注来源：JD 原文 / 我的经历 / 通用求职判断。涉及公司近况、薪资等最新信息，请写"需要补充公开信息后再判断"。
```

## 评测重点

- 对比表是否忠实于各 JD 原文，没有混淆或臆造要求。
- 排序依据是否具体到"哪条 JD 要求对应我哪条经历"，而不是感觉分。
- 差异化调整是否只动表达重点，没有为不同岗位捏造不同版本的经历。
- 匹配度都很低时，是否诚实建议先补齐能力，而不是硬排出一个"最优"。

## 和其他模块的关系

- 适合放在主流程之前用：先决定投哪个，再对选中的 JD 走 01-06 完整优化流程。
- 网页表单版（`index.html`）第 5 步"进阶工具"支持粘贴额外 JD（用 `---` 分隔）一键拼装。
- 对结果不满意可以用 `07-feedback-refine.md` 调整。
