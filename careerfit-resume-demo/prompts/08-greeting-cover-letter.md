# 08 打招呼语 / 求职信生成 Prompt

## 使用场景

简历优化完成后，用户在实际投递时还差"临门一脚"的沟通材料：

- Boss 直聘 / 实习僧 / 牛客等平台的**打招呼语**（30-80 字，决定 HR 是否点开简历）。
- 邮件投递或外企/部分平台需要的**求职信 / 自我介绍段落**（150-300 字）。

这个模块借鉴了 Resume-Matcher 和 JadeAI 的 Cover Letter Generator 思路，但针对中文求职场景做了调整：国内校招/实习场景里，平台打招呼语的使用频率远高于正式 Cover Letter，所以两种都提供，且默认先生成打招呼语。

## 设计原则

- 只基于用户真实经历和 JD 内容生成，不编造实习、项目或数据。
- 打招呼语必须短：30-80 字，1-2 个最强匹配点 + 1 个明确诉求（求简历被查看/约面），拒绝套话堆砌。
- 求职信控制在 150-300 字，结构为：匹配点 -> 证据 -> 意愿，不写"贵公司"式空洞恭维超过一句。
- 每个版本都标注引用了用户哪条经历，方便用户核对真实性。
- 生成 2-3 个风格版本（稳重版 / 亮点前置版 / 简短直接版）供用户挑选，不替用户决定。
- 如果用户经历和岗位差距明显，提示"打招呼语无法弥补硬性差距"，不鼓励夸大。

## Prompt 模板

```text
你是一名熟悉国内求职平台（Boss 直聘、实习僧、牛客）和邮件投递习惯的求职沟通顾问。

请根据下面的目标岗位 JD 和我的真实背景，帮我生成求职沟通材料。

重要原则：
1. 只能使用我真实提供过的经历，不允许编造实习、项目、数据或荣誉。
2. 每个版本后面用一行标注"引用经历：xxx"，说明用到了我背景里的哪条信息。
3. 不写夸张承诺（如"我一定能带来 xx 增长"），只陈述可验证的事实和意愿。
4. 如果我的背景和岗位差距明显，请直接说明，并建议我优先补齐什么，而不是用话术掩盖。

【目标岗位 JD】
<<<
粘贴 JD
>>>

【我的个人背景】
- 学历/专业：
- 目标岗位：
- 过往实习：
- 项目经历：
- 校园/社团经历：
- 内容/运营经历：
- 会使用的工具：
- AI 工具使用经历：

请输出：

## 1. 平台打招呼语（Boss 直聘 / 实习僧风格）

生成 3 个版本，每个 30-80 字：
- 版本 A（稳重版）：学历/专业 + 最相关经历 + 求查看简历
- 版本 B（亮点前置版）：用最强的一条匹配经历开头
- 版本 C（简短直接版）：一句话说清"我是谁 + 为什么匹配"

每个版本标注：引用经历、适合什么情况用。

## 2. 邮件求职信 / 自我介绍段落

生成 2 个版本，每个 150-300 字：
- 版本 A（标准版）：开头点明应聘岗位 -> 2 个匹配点及证据 -> 表达意愿和可到岗时间占位
- 版本 B（针对性版）：围绕 JD 里最核心的一条要求展开

每个版本标注：引用经历、需要我自己补充的信息（用【】标出，例如【可实习时长】）。

## 3. 真实性检查

列出以上版本中所有引用的经历事实，让我逐条确认是否属实。如果某条是从我的描述推断而来（不是我明确写过的），请标注"推断，请确认"。
```

## 评测重点

- 打招呼语是否控制在字数内，且包含具体匹配点而不是"我很感兴趣、我学习能力强"这类空话。
- 是否出现了用户未提供的机构名、数据、项目。
- 真实性检查部分是否完整列出了引用事实。
- 对明显不匹配的岗位，是否诚实提示而不是硬写。

## 和其他模块的关系
