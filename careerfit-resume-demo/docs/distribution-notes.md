# 分发说明

这个文件用于说明工具包如何发布、下载和分享。公开仓库只保留必要分发信息，避免分发话术淹没产品本身。

## 推荐公开路径

GitHub 作为完整工具包的主入口。小红书等内容平台只展示效果图、解释使用场景，并给一个网页表单链接。

网页表单版（`index.html`，部署在 GitHub Pages）是现在推荐给小红书用户的主路径，比"下载仓库找文件复制"门槛低很多：

```text
小红书效果图
↓
网页表单链接（GitHub Pages）
↓
填 JD + 个人经历，点"生成 Prompt"
↓
点"复制 Prompt"
↓
粘贴到自己平时用的任意 AI 工具（豆包、千问、DeepSeek、Kimi、ChatGPT、Claude 都可以）
↓
获得 A 简历优化结果 + B 求职建议
```

想深入研究 Prompt 设计、看行业 Demo、查 JD 测试集的用户，再引导去 GitHub 仓库主页。

## GitHub 发布检查

- `README.md` 聚焦说明项目是什么、适合谁、怎么用。
- `QUICKSTART.md` 作为小白用户入口。
- `docs/copy-paste-simple-prompt.md` 作为最短使用路径。
- `examples/`、JD 测试、评测方法、失败案例用于证明产品过程。
- 不上传真实用户简历、手机号、微信号、邮箱、学校、公司等隐私信息。
- 仓库已使用 MIT License。

## 小白使用方式

给完全不懂 GitHub、不想下载任何东西的用户（网页表单版，推荐）：

```text
1. 点开网页链接（手机浏览器也能打开）。
2. 填目标岗位 JD 和自己的真实经历。
3. 点"生成 Prompt"。
4. 点"复制 Prompt"。
5. 打开自己平时用的 AI（豆包、千问、DeepSeek、Kimi、ChatGPT、Claude 都可以），粘贴，发送。
6. 得到简历优化结果 + 求职建议。
```

给想深入研究、下载完整工具包的用户：

```text
1. 打开 GitHub 仓库。
2. 点击 Code。
3. 点击 Download ZIP。
4. 解压文件夹。
5. 打开 QUICKSTART.md。
```

### 标题备选

### 正文草稿

```text
最近我发现一个很常见的问题：

很多同学不是没有经历，而是不会把经历写成岗位看得懂的语言。

比如：
- 做过课程报告，但不知道怎么写成"数据分析能力"
- 运营过小红书，但只会写"发布内容"
- 用过AI工具，但不知道怎么写得专业又不夸张
- 看JD时一堆要求，看完还是不知道自己适不适合投

所以我做了一个免费小工具：CareerFit Resume Demo

用法很简单，方便易用：
1. 打开网页（链接放评论区/主页）
2. 填你的目标岗位JD + 真实经历
3. 点一下"生成Prompt"，网页会自动帮你把内容拼成一份完整、结构化的求职分析Prompt
4. 复制这段Prompt，粘贴到你平时习惯用的AI里就行——豆包、千问、DeepSeek、Kimi、ChatGPT、Claude，随便哪个都可以，不绑定任何一个

AI会给你两部分结果：
A. 简历优化结果：岗位要求提取、履历事项挖掘、匹配度打分、优化后的简历表达
B. 求职建议：能力差距、1-3个候选补齐项目、面试风险提醒（这部分不直接放进简历）

全程不用下载、不用注册、不会收集你的任何数据，所有内容都在你自己浏览器里生成。

想深入研究的同学，GitHub仓库里还有完整Prompt工具包、8个行业方向的示例Demo、真实JD测试记录，链接也在评论区。

这个工具不会帮你编经历。如果你没做过某件事，它只会提示"需要补充证据"或"建议通过项目补齐"，不会把没有的经历写成真的——毕竟简历最后还是要经得起面试追问。

如果你想让我优先做哪个岗位方向的示例，欢迎评论告诉我。
```

### 注意事项

- 封面图是效果示意图，不是真实用户简历或真实 AI 对话截图，正文要注明。
- 不说"保证拿offer"、"自动通过ATS"。

- 不引导私信、加微信、进群领取。

## 不建议宣传

- 不说保证拿 offer 或保证获得面试。
- 不说自动通过 ATS。

- 不要求用户发送完整简历或隐私信息。
- 不使用加私域、进群、加联系方式等强引流话术。

## 适合收集的反馈

优先收集低敏反馈：

- 目标岗位方向
- 使用的 AI 工具
- 输出是否看得懂
- 哪一步最卡
- 是否发现 AI 编造或夸大
- 优化后的简历表达是否更清楚
