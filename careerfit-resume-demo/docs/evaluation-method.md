# Evaluation Method

## 为什么需要评测

简历优化类工具很容易看起来有用，但实际输出可能存在以下问题：

- 漏掉岗位里的关键要求
- 把不重要的信息误判成核心技能
- 匹配分数虚高
- 简历建议过于泛泛
- 生成不存在的经历

所以本项目需要建立简单、可复查的评测方法。

## 测试集

第一阶段已收集 10 份真实 JD（见 `docs/jd-test-set.md`），覆盖以下方向：

- AI/大模型产品实习（JD 001）
- 数据分析（JD 002）
- 新媒体/内容运营（JD 003）
- 产品/电商产品（JD 004）
- 市场/品牌营销（JD 005）
- 互联网产品/运营（JD 006）
- 电商运营（JD 007）
- 金融/商业分析（JD 008）
- HR/行政（JD 009）
- 管培生（JD 010）

8 个岗位方向均已有至少 1 份 JD 记录。其中 JD 006、009、010 的内容来自搜索引擎摘要或第三方转载页面，非企业官方一手页面存档，详见 `docs/jd-test-set.md` 的"本轮扩充说明"。

对应的完整简历优化 Demo（见 `examples/`）目前覆盖 7 个方向：AI/大模型产品实习、数据分析、新媒体/内容运营、互联网产品/运营、电商运营、金融/商业分析、HR/行政/管培生。市场/品牌营销方向暂时只有 JD 记录，还没有单独的 Demo。

## 人工标注

每份 JD 需要人工标注：

- 核心职责
- 必备技能
- 加分技能
- 软技能要求
- 简历关键词
- 可补齐项目方向

## 错误类型

| 错误类型 | 说明 |
| --- | --- |
| 漏提取 | JD 里明确提到的重要技能没有被提取 |
| 误提取 | 把不重要或不存在的内容当作核心要求 |
| 分类错误 | 把技能放错类别，例如把软技能放进硬技能 |
| 匹配虚高 | 用户经历不足，但系统给出过高匹配分 |
| 建议泛化 | 输出建议太空泛，无法执行 |
| 经历幻觉 | 生成用户没有提供过的经历 |

## 迭代记录

每次修改 Prompt 后，需要记录：

- 修改前问题
- 修改内容
- 修改后输出变化
- 是否解决原问题

第 1 轮迭代记录见 `docs/failure-cases.md` 的 Case 001-004，覆盖增强版简历表达误用风险、正式岗匹配虚高、模拟用户背景复用导致建议泛化、JD 测试集来源集中四类问题。
