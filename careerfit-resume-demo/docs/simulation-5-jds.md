# 5 个公开 JD 模拟测试

## 说明

本文件用于测试当前工具包的新逻辑：

```text
JD 要求提取
↓
个人履历事项挖掘
↓
输出 A：优化后的简历
↓
输出 B：JD 匹配建议
```

注意：

- 以下 JD 均来自公开招聘页面或接口。
- 部分岗位是正式岗，不是实习岗。本文件只用于提炼岗位要求和模拟简历优化，不代表建议小白直接投递正式岗。
- 模拟用户背景为同一个“小白用户”，方便观察不同 JD 下的输出差异。

## 模拟用户背景

```text
本科商科/管理类专业，无正式大厂实习。
做过课程市场调研报告，负责问卷整理、Excel 图表和汇报。
运营过公开内容账号，内容为学习经验和生活方式，积累一定关注基础，有单篇高互动内容。
参与过社团活动策划和公众号推文编辑，负责资料整理、文案排版和活动复盘。
会使用 ChatGPT、Kimi、Canva、Excel，SQL 基础较弱。
目标：希望投产品/运营/数据分析/AI 相关实习岗位。
```

## 本轮模拟结论

### 工具逻辑是否跑通

已跑通。

5 个 JD 都能按照以下结构输出：

1. JD 要求提取
2. 履历事项挖掘
3. 输出 A：优化后的简历
4. 输出 B：JD 匹配建议

### 发现的问题

| 问题 | 说明 | 迭代建议 |
| --- | --- | --- |
| 正式岗 JD 门槛偏高 | 多数公开大厂 JD 是正式岗，年限要求高 | 在 Demo 中明确“用于反推实习准备要求” |
| 简历优化容易写太满 | 增强版容易把未完成项目写成已完成 | 保留“完成前不能写进简历”字段 |
| 不同岗位共用同一用户背景会有偏差 | 同一履历适合运营/内容，但不一定适合数据/产品 | 后续每类岗位设计不同模拟用户 |
| JD 来源集中于腾讯 | 便于获取结构化文本，但行业覆盖不够广 | 后续补 Boss、牛客、实习僧、公司官网公开 JD |

### 下一步建议

- 增加 5 个非腾讯来源 JD，提升样本多样性。
- 给每个主流行业单独设计一个模拟用户背景。
- 把本文件拆成 5 个独立 Demo，便于对外分享。
