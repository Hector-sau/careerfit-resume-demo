# Failure Cases

这个文件用于记录 Prompt 输出失败的真实案例，以及项目在测试过程中发现的方法论问题。

Case 001-004 整理自 `docs/simulation-5-jds.md` 中 5 份 JD 模拟测试记录的"发现的问题"部分，以及后续补充 JD 006-010 时暴露的数据来源问题。新案例请复制文末的"记录模板"继续填写。

## Case 001：增强版简历表达存在被误用为已完成经历的风险

目标岗位：

```text
AI产品经理-AI生成游戏（JD 001，用于反推 AI 产品实习准备要求）
```

用户背景：

```text
ChatGPT 辅助课程报告、公开内容平台 一定关注基础，无 AI 产品项目经验
```

失败类型：

- [ ] 漏提取
- [ ] 误提取
- [ ] 分类错误
- [ ] 匹配虚高
- [ ] 建议泛化
- [x] 经历幻觉（风险）

问题描述：

```text
04-resume-rewrite Prompt 早期版本只输出"保守版/增强版"两个简历表达版本，
没有在结构上强调"增强版需要完成对应项目后才能使用"，用户容易直接复制增强版
内容当作已完成经历投递简历，构成经历幻觉风险。
```

修改前输出：

```text
增强版 bullet 直接以陈述句呈现（例如"设计了 JD 解析与匹配评分 Prompt 模块……"），
没有任何醒目提示区分"已完成"和"计划完成"。
```

Prompt 修改：

```text
04-resume-rewrite.md 增加"真实性检查"章节；06-career-advice.md 的"可补齐项目建议"
新增"完成前不能写进简历的内容"和"完成后可以写进简历的表达"两个必填字段。
```

修改后输出：

```text
每个补齐项目建议都清晰拆分成"完成前不能写"和"完成后可以写"两部分，
用户能直接判断当前能不能使用某条表达。
```

结论：

```text
问题已通过 Prompt 结构调整缓解，但仍依赖用户自觉核对，不能完全依赖 Prompt 自我约束。
后续可以考虑在简单版 Prompt（copy-paste-simple-prompt.md）里也加入同样的强提示。
```

## Case 002：正式岗 JD 直接打分容易出现匹配虚高

目标岗位：

```text
企业微信-数据分析师（JD 002，正式岗，用于反推数据分析实习准备要求）
```

用户背景：

```text
课程市场调研 + Excel 基础，不会 SQL
```

失败类型：

- [ ] 漏提取
- [ ] 误提取
- [ ] 分类错误
- [x] 匹配虚高
- [ ] 建议泛化
- [ ] 经历幻觉

问题描述：

```text
如果直接把正式岗 JD 喂给 02-fit-analysis Prompt 而不做额外说明，AI 有可能
默认按"实习生"的宽松标准打分，导致匹配分偏高，掩盖了正式岗实际的年限和
深度要求带来的门槛差距。
```

修改前输出：

```text
未特别说明岗位性质时，匹配度分析容易忽略"这是正式岗"这一前提，
直接给出偏乐观的分数和"建议投递"结论。
```

Prompt 修改：

```text
在 Demo 文档和 jd-test-set.md 里明确标注每条 JD 是否为正式岗；
01-jd-parser.md 的"岗位基本判断"部分要求 AI 先输出"岗位级别判断"，
再进行后续的能力提取和匹配分析。
```

修改后输出：

```text
JD 解析结果会先给出"岗位级别判断"，匹配度分析会注明
"该分数基于反推实习准备要求，不代表可直接对标正式岗招聘标准"。
```

结论：

```text
问题得到部分缓解。正式岗 JD 仍建议优先用于提取能力要求，
不直接用于精确匹配打分，这一限制已写入相关 Demo 说明中。
```

## Case 003：多个 Demo 复用同一模拟用户背景，说服力不足

目标岗位：

```text
docs/simulation-5-jds.md 中的 5 个模拟 JD（产品、数据分析、内容运营、电商产品、市场营销）
```

用户背景：

```text
统一的"小白用户"背景（本科商科、无正式实习、公开内容平台 一定关注基础、Excel + ChatGPT 基础）
```

失败类型：

- [ ] 漏提取
- [ ] 误提取
- [ ] 分类错误
- [ ] 匹配虚高
- [x] 建议泛化
- [ ] 经历幻觉

问题描述：

```text
为了快速验证 Prompt 逻辑，5 个 JD 模拟测试复用了同一个用户背景，
导致部分方向（例如市场营销）的"履历事项挖掘"结果显得牵强，
反复把同一段内容平台运营经历套进不同岗位话术里，用词高度相似，说服力偏弱。
```

修改前输出：

```text
同一段"公开内容账号运营，一定关注基础"的经历，在电商产品、市场营销、数据分析
场景下被反复关联到不同能力标签，缺少针对性差异。
```

Prompt 修改：

```text
不是 Prompt 本身的修改，是 Demo 制作方法论的调整：
后续新增的 4 个正式 Demo（互联网产品运营、电商运营、金融商业分析、
HR/管培生）改为每个方向单独设计更贴合的模拟用户背景，不再复用统一背景。
```

修改后输出：

```text
新 Demo 的模拟背景更贴合各自方向，例如电商运营 Demo 加入了社团义卖定价经历，
HR Demo 加入了社团组织部部长和纳新排班经历，减少了跨方向的重复感。
```

结论：

```text
已通过为每个方向单独设计背景解决。后续新增岗位方向的 Demo 也应遵循这个原则，
不能为了省时间直接复用已有背景。
```

## Case 004：JD 测试集来源集中，行业覆盖广度有限

目标岗位：

```text
全部 10 份 JD 测试集（docs/jd-test-set.md）
```

用户背景：

```text
不适用——这是测试集数据来源层面的问题，不是单条 Prompt 输出问题
```

失败类型：

- [ ] 漏提取
- [ ] 误提取
- [ ] 分类错误
- [ ] 匹配虚高
- [x] 建议泛化（间接导致后续能力标注可能有偏差）
- [ ] 经历幻觉

问题描述：

```text
JD 001-005 全部来自腾讯招聘公开接口，虽然 JD 006-010 补充了字节跳动、
以及经国家大学生就业服务平台（ncss.cn）转载的其他公司岗位，
但整体样本仍以互联网大厂和第三方转载页面为主，缺少猎聘、BOSS 直聘等
平台的一手数据，行业覆盖广度依然有限，招聘方也以互联网/教育/生物医药
等有限行业为主。
```

修改前输出：

```text
JD 001-005 全部来源标注为"腾讯招聘公开岗位接口"，来源单一。
```

Prompt 修改：

```text
不涉及 Prompt 本身修改，属于测试集迭代方向。已在 docs/jd-test-set.md
的"本轮扩充说明"中如实记录这一局限，并注明部分链接可能因岗位下线失效。
```

修改后输出：

```text
JD 006-010 引入了字节跳动官网、ncss.cn 转载的智联招聘/实习僧/高校就业网
等多个来源，来源多样性有提升，但仍以转载页面为主，非企业官方一手页面。
```

结论：

```text
未完全解决。下一轮迭代建议补充非 ncss 转载的一手 JD 来源，
并覆盖当前测试集里还比较薄弱的行业（例如金融、制造业）。
```
