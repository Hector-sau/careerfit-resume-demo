# Product Brief

## 项目定位

CareerFit Resume Demo 是一个面向求职小白的简历优化 Prompt 工具包。它帮助用户把目标岗位 JD 和个人经历输入到 AI 工具中，先挖掘个人履历事项并生成优化后的简历表达，再单独输出岗位匹配分析、技能差距和候选补齐项目。

补齐项目不由系统替用户决定，而是提供 1-3 个候选方向。用户根据兴趣、时间、技能基础和求职目标自行选择。

## 不是做什么

- 不是自动投递工具
- 不是保证拿 offer 的工具
- 不是替用户编造经历的简历生成器
- 不是一开始就依赖 API 成本的网页产品

## 解决的问题

很多学生和转方向求职者有经历，但不会判断这些经历和岗位是否匹配，也不会把经历写成岗位看得懂的简历语言。

本项目通过结构化 Prompt 和 Demo，帮助用户完成：

- 看懂 JD
- 判断岗位要求
- 识别自己缺什么
- 把真实经历转化成更清晰的简历表达
- 单独获得基于 JD 的投递建议、能力差距和候选补齐项目
- 看到每条关键建议的信息来源和判断依据
- 对任意 AI（不限定工具）生成的结果不满意时，粘回结果并勾选问题类型，生成一份带具体修正指令的调整 Prompt，不用重新描述一遍需求

## 信息来源

MVP 阶段不做爬虫，也不自动抓取招聘平台数据。求职建议主要来自：

- 用户粘贴的目标岗位 JD
- 用户提供的真实个人经历
- 已优化的简历表达
- 项目内公开 JD 测试集
- 通用求职判断和面试经验推断

涉及公司近况、岗位是否仍开放、薪资、招聘数量、行业趋势等实时信息时，需要用户补充最新公开链接或自行核实。

## MVP 形态

第一版是 GitHub Prompt 工具包，不直接调用模型 API。用户复制 Prompt 到自己常用的 AI 工具中使用。

## 后续形态

第二版是 GitHub Pages 静态网页表单（`index.html`），已经实现。用户填写 JD 和个人经历后，网页在浏览器本地自动拼装出可复制 Prompt，不接后端、不接模型 API、不收集数据，只是把"找文件复制粘贴"简化成"填表单点按钮"，专门用来降低小红书用户的使用门槛。

第三版再考虑接入模型 API，实现一键生成报告。这一步会产生真实成本和滥用风险，计划在网页表单版收集到真实使用反馈、确认有稳定需求后再评估，不提前投入。
